# Optional Extensions

Extensions ranked by effort. Core project is complete; these are future work.

---

## Low effort / high value

### LoRA adapter
Fine-tune GeoRSCLIP's attention weights via `peft` library instead of adding a separate `ProjectionHead`.
GeoRSCLIP natively supports LoRA, Pfeiffer, Prefix Tuning, UniPELT.

- **Why:** ProjectionHead (current PEFT) memorises train-AOI texture and collapses on val/test (0.428 → 0.042 mAP). LoRA modifies backbone weights — potentially transfers better across AOIs.
- **How:** Add `--adapter lora` flag to `src/train.py`; apply LoRA via `peft.get_peft_model(encoder.model, lora_config)`; compare cross-split table against ProjectionHead.
- **Effort:** ~1 day.

### NDVI ablation
`color_mode='ndvi'` is already implemented in `DENNpyDataset` but never benchmarked.

- **Why:** NDVI = (NIR−R)/(NIR+R) — directly encodes vegetation density. May outperform NRG for vegetation-change queries; easy ablation point.
- **How:** `python -m scripts.run_pipeline --color-mode ndvi --skip-train --eval-splits train val test`
- **Effort:** ~30 minutes (just run it; add one row to REPORT.md §7.3).

### Hosted demo
Gradio app already built. Deploy to HuggingFace Spaces.

- **Why:** Shareable link; zero extra code.
- **How:** `gradio deploy` or push repo to a HuggingFace Space with `app.py` as entry point; set `--root` to a bundled tiny fixture or streamed DEN subset.
- **Effort:** ~2 hours (HF account, Space setup, fixture bundling).

## Medium effort

### LLM-generated captions for PEFT supervision
Replace template captions (`"agriculture replaced by impervious surface"`) with richer descriptions generated by an LLM (GPT-4o-mini, Mistral, etc.).

- **Why:** Current weak supervision is coarse — two class names only. Richer captions closer to query distribution may reduce PEFT overfitting.
- **How:** For each `PairLabel`, prompt LLM with class names + change fraction; cache results; pass to `train.py` via `text_caption_for_pair()` override.
- **Effort:** ~2 days.

### QFabric full pipeline
Run full benchmark on QFabric (450k change polygons, 6 change types, 9 change status classes, 504 locations).

- **Why:** Tests dataset-agnosticism claim. Different change taxonomy and annotation style from DEN.
- **How:** Download full QFabric via HuggingFace `EVER-Z/QFabric_mt_images_1024`; `QFabricDataset` + `src/queries/qfabric.py` already stub-present; flesh out `get_pair_label` and query set.
- **Effort:** ~3 days (data download + query set design + benchmark run).

### fMoW-Sentinel pipeline
Multi-temporal Sentinel-2 imagery, 63 land-use categories, worldwide coverage.

- **Why:** Entirely different sensor (10m Sentinel-2 vs 3m Planet) and scale; stress-tests encoder generalisation.
- **How:** Download via HuggingFace `jonathan-roberts1/fMoW-Sentinel`; implement `FMoWDataset(TemporalDataset)` in `src/datasets/fmow.py`; register; add query set.
- **Effort:** ~3–5 days.

### Geographic filtering in UI
Use `data/DynamicEarthNet/aoi_metadata.json` (WGS84 bboxes per AOI) to add a region-filter widget to the Gradio app.

- **Why:** Lets users constrain retrieval to a geographic area (e.g., "only Europe"). Adds spatial search dimension without touching retrieval logic.
- **How:** Add a map widget or dropdown of regions in `src/app.py`; pre-filter `pair_ids` by bbox overlap before scoring.
- **Effort:** ~1 day.

## High effort / research-grade

### SAR (Sentinel-1) fusion
Add S1 SAR backscatter as an additional input channel alongside Planet optical imagery.

- **Why:** SAR penetrates cloud cover; sensitive to soil moisture, surface roughness, vegetation structure. New modality = new signal CLIP never sees from optical alone.
- **Blockers:** S1 paths in `torchgeo_meta/splits.csv` point inside the 525 GB TUM Mediatum dump — no clean separate download. CLIP/GeoRS/RemoteCLIP trained on optical only; SAR backscatter is noise to them without a SAR encoder (e.g., SAR-JEPA, CROMA) or SAR-to-optical colorisation step.
- **How:** Download TUM Mediatum; implement `DENSARDataset`; fuse SAR + optical embeddings (late fusion: concatenate, then project); or use CROMA dual-encoder directly.
- **Effort:** 2–4 weeks.

### Temporal attention encoder
Encode all 24 monthly frames jointly using a temporal transformer instead of bi-temporal pairs.

- **Why:** Current approach treats each (T1, T2) pair independently. A temporal model sees the full time series — can detect gradual drift, seasonal cycles, and abrupt events separately.
- **How:** Replace per-frame CLIP encode + difference with a video-CLIP or TimeSformer backbone over the 24-frame sequence; adapt `PairKey` to `SequenceKey`; update `embeddings.py` cache format.
- **Effort:** 1–3 weeks.

### Retrieval re-ranking
Use spatial/temporal coherence to re-rank initial cosine-similarity results.

- **Why:** Nearby AOIs or temporally adjacent pairs likely share change patterns. A re-ranker exploiting this can improve Recall@K without retraining.
- **How:** After top-K retrieval, score candidate pairs by geographic proximity (haversine on bbox centroids from `aoi_metadata.json`) and temporal distance; combine with cosine score via learned or heuristic weights.
- **Effort:** ~3 days.

### Human relevance judgements
Replace LULC-derived pseudo-labels with human-annotated query relevance for a proper IR-style benchmark.

- **Why:** Current `PairLabel` relevance is a heuristic over class-transition rules. Human judges directly assess whether a retrieved pair matches the query intent — ground truth for a retrieval system.
- **How:** Export top-K results per query to an annotation interface (Label Studio, Argilla); collect binary/graded relevance; recompute mAP/nDCG against human labels.
- **Effort:** 1–2 weeks (annotation platform setup + annotation time).
